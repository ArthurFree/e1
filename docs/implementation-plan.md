# 实现步骤

## 第 1 阶段：工程与基础数据

1. 初始化独立的 React + Vite + TypeScript 项目、代码规范、测试和环境变量模板。
2. 建立主题令牌、全局布局和响应式壳层。
3. 定义领域类型与仓储接口，完成 IndexedDB schema、初始化数据和迁移策略。
4. 实现工作区、页面树、标题编辑、自动保存和主题持久化。

完成标准：可离线打开预置知识库，创建及编辑文档标题后刷新仍存在。

## 第 2 阶段：编辑器核心

1. 配置 Tiptap 开源扩展包与中文欢迎文档 JSON。
2. 实现编辑器保存适配器、图片处理和 Markdown 输入规则。
3. 加入 `/` 命令、选区浮动工具栏、链接与颜色/高亮菜单。
4. 加入任务列表、目录、Emoji、@ 提及、图片、代码块和数学公式。

完成标准：示例文档可完整编辑，所有变更可恢复并可用键盘完成主要操作。

## 第 3 阶段：高级块交互

1. 添加 DragHandle、块拖动、复制、删除、转换和格式重置菜单。
2. 实现表格操作：行列增删、表头、合并/拆分、拖动和排序。
3. 实现撤销/重做、顶部控制栏和深浅色细节。
4. 对齐模板的字级、行高、菜单尺寸、边框、阴影和动画。

完成标准：模板参考中的关键交互均有开源等价实现，1440 × 900 主画面结构达标。

## 第 4 阶段：笔记管理能力

1. 实现页面树拖放、标签、全局搜索与回收站。
2. 实现 Markdown 导入、导出和导入失败提示。
3. 完善空状态、加载状态、异常状态和数据损坏恢复。

完成标准：用户可不用任何外部服务管理完整本地知识库。

## 第 5 阶段：AI 与质量收尾

1. 实现 AI 设置、OpenAI-compatible provider 与安全校验。
2. 实现 `/ask ai`、选区改写、总结、润色与结果预览/应用。
3. 编写单元、交互和端到端测试；建立 1440 × 900 截图基线并处理视觉偏差。
4. 完成 README、运行说明、隐私说明与发布前检查。

完成标准：未配置 AI 安全降级；配置有效兼容服务后能将结果应用至文档；全部测试通过。

## 实施顺序约束

- IndexedDB 仓储与页面领域模型先于编辑器保存逻辑。
- 编辑器命令注册先于命令菜单、浮动工具栏和块菜单。
- 表格、拖拽和 AI 在基础编辑器稳定后再引入，减少状态耦合。
- 每阶段完成后才更新视觉基线，避免将未完成界面固定为标准。
