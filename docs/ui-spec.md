# UI 与视觉规格

> 本文保留项目初始视觉基线。R001 完成后的下一轮完整 UI 设计、令牌、组件、响应式和实施要求见 [`requirements/R002-yuque-ui-design-system.md`](./requirements/R002-yuque-ui-design-system.md)，实际实现以 R002 为准。

## 基准与原则

- 基准视口：1440 × 900。
- 文案：简体中文；视觉比对关注结构、留白、色彩、控件位置、尺寸和状态，不对英文原模板逐字比对。
- 风格：参考 Notion-like 模板的白色画布、细边框、低对比灰阶、圆角浮层、轻阴影和密度。

## 页面结构

```text
应用壳
├── 工作区侧栏：知识库切换、搜索、设置、回收站
├── 文档树侧栏：可拖动宽度、目录树、创建入口
└── 主栏
    ├── 顶部栏：撤销、重做、主题切换、文档操作
    ├── 文档头：图标、标题、标签、导入/导出
    └── 编辑画布：最大内容宽度、块操作和浮动菜单
```

## 必须覆盖的视觉状态

1. 中文欢迎文档：标题、引用块、代码块、图片占位、任务列表、示例表格和目录。
2. 浅色与深色主题。
3. `/` 命令菜单打开且有键盘高亮项。
4. 文本被选中时的浮动格式工具栏。
5. 块悬停时的拖拽把手及其上下文菜单。
6. 表格行或列的操作菜单。
7. 搜索弹窗、标签选择器和回收站状态。
8. 未配置与已配置 AI 的命令状态。

## 响应式规则

- 大于等于 1024px：显示完整双侧栏和主编辑区。
- 768px 至 1023px：工作区侧栏收窄，文档树可覆盖式展开。
- 小于 768px：侧栏切换为抽屉，编辑区使用较小边距，浮动菜单避免溢出。

## 可访问性

- 所有图标按钮提供 `aria-label` 和可见的焦点状态。
- 菜单可用方向键、Enter 和 Escape 操作。
- 主题色满足文本对比度要求；不可只依赖颜色表达任务或选中状态。
