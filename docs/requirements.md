# 产品需求

## 目标

实现一个可离线使用的 Web 笔记应用。编辑体验和视觉层级参考 Tiptap Notion-like 模板，应用层补充知识库、页面树、搜索、标签、回收站和 Markdown 导入导出。

## 用户与核心场景

个人用户在浏览器中创建知识库、组织文档并进行富文本笔记。用户可在离线状态下编辑，刷新页面后继续使用；可通过搜索、标签和页面树定位内容；可选配兼容 OpenAI API 的服务协助写作。

## 功能需求

### 工作区与文档组织

- 创建、重命名和切换知识库。
- 以树形结构创建、重命名、移动、折叠和删除文件夹与文档。
- 将删除内容放入回收站，支持恢复和永久删除。
- 为文档添加、移除和筛选标签。
- 通过全局搜索按标题和正文查找文档。

### 编辑器

- 支持标题、段落、引用、代码块、分隔线、项目列表、编号列表、任务列表、图片、链接、Emoji、@ 提及、文本颜色、高亮、对齐和数学公式。
- 支持表格编辑：新增/删除行列、表头、合并/拆分单元格、行列拖动和排序。
- 支持目录，依据文档标题自动更新并可跳转。
- 提供 Markdown 快捷输入、`/` 命令菜单、文本选区浮动工具栏、块拖拽把手与块菜单。
- 支持撤销、重做、深浅色主题切换和自动保存。

### 导入、导出与 AI

- 导入 Markdown 文件并转换为可编辑内容；将文档导出为 Markdown 文件。
- 设置页可配置 OpenAI-compatible endpoint、模型名和 API key。
- 配置完成后，`/ask ai`、选区改写、润色和总结可将结果插入或替换当前选区。
- 未配置 AI 时，不请求网络，并以禁用状态说明配置入口。

## 非功能需求

- 首屏为可用的本地应用；不依赖账号或应用自建后端。
- 所有本地数据使用 IndexedDB；浏览器刷新、关闭再打开后可恢复。
- API key 不写入页面、日志或应用服务器；只存浏览器本地。
- 1440 × 900 下以高还原度复刻参考模板的结构、间距、颜色、控件和交互状态；窄屏保持可用。
- 关键编辑操作具备键盘可达性，图标按钮提供可访问名称。

## 范围边界

首版不做注册/登录、跨设备同步、实时协作、评论、权限、支付，也不使用 Tiptap Pro 的专有 AI 或高级 UI 组件。AI 使用自建的开源等价 UI 和通用 HTTP 适配层。

## 验收标准

- 用户可完整创建、编辑、组织、搜索、删除和恢复本地笔记。
- 预置中文示例文档覆盖参考模板的欢迎信息、任务清单、表格、目录和提示块。
- 编辑内容、标签、树结构、主题和设置在刷新后保持不变。
- 在未配置 AI 的浏览器中零 AI 网络请求；配置有效服务后可获得并应用 AI 输出。
- 指定编辑器状态的 1440 × 900 截图通过视觉回归阈值。
