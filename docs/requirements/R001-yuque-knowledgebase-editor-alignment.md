## 1. 背景

当前项目已经具备工作区、文档树、文件夹、标签、搜索、回收站、Markdown 导入导出、块编辑、表格、目录、AI 助手和本地持久化。与本机语雀对比后，基础写作能力已经可用，但知识库信息架构仍偏“文件浏览器”，编辑器的大部分能力隐藏在 `/` 命令、选区工具栏和块菜单中。

本机语雀呈现了更明确的四级组织关系，并区分全局工作台首页与单个知识库首页：

```text
个人空间
├── 开始/工作台首页
│   ├── 快速创建
│   ├── 模板与 AI 入口
│   └── 编辑/浏览记录
└── 知识库
    ├── 知识库首页
    ├── 目录
    │   ├── 分组
    │   │   └── 文档
    │   └── 文档
    └── 文档编辑区
```

编辑区还提供常驻格式工具栏、保存状态、字数、大纲、版本/分享入口。第一轮不复制语雀的云端平台能力，而是在当前本地优先边界内补齐对个人知识库最有价值的结构和编辑体验。

## 2. 目标

### 2.1 产品目标

- 增加全局“开始”首页，集中提供快速创建、最近活动、筛选、内置模板和 AI 写作入口。
- 将 `Workspace` 明确呈现为“知识库”，具有图标、描述、收藏、统计、首页和目录。
- 将当前“文件夹”升级为语义明确的“分组”，继续支持任意层级、折叠和拖拽排序。
- 将最近编辑、最近浏览和收藏提升为跨知识库的全局视图，让用户不必先进入某个知识库寻找文档。
- 为编辑区增加语雀式常驻工具栏，让常用格式无需先选中文本或记忆命令。
- 增加保存状态、字数统计和本地版本历史，提升长文编辑的安全感。
- 支持本地附件块与代码语言高亮，补齐常见技术文档场景。

### 2.2 成功指标

- 新用户可在不使用 `/` 命令的情况下，通过可见工具栏完成标题、文本格式、列表、对齐、颜色、链接、表格和图片操作。
- 应用启动后，用户无需先进入知识库即可看到最近编辑文档并在一次点击内打开。
- 用户可从开始首页在两步内完成新建文档、新建知识库、从模板创建或启动 AI 写作。
- 旧数据库升级后，知识库、原文件夹层级、文档、标签和正文完整保留。
- 自动保存过程有明确状态，保存失败不会被误显示为已保存。
- 用户可查看最近版本并恢复；恢复前自动保存当前版本，避免二次丢失。

## 3. 范围

### 3.1 本需求包含

1. 全局“开始”首页及快速创建区域。
2. 知识库元数据、知识库首页和目录结构。
3. 文件夹到分组的兼容迁移。
4. 跨知识库的最近编辑、最近浏览和收藏视图。
5. 内置本地模板中心与 AI 写作快捷入口。
6. 常驻编辑工具栏与统一命令注册。
7. 标题 4–6、字号、缩进、代码语言与清除格式。
8. 字数、保存状态和本地版本历史。
9. 本地附件块。
10. 对应的数据迁移、自动化测试和视觉回归。

### 3.2 本需求不包含

- 注册登录、云同步、公开分享、多人实时协作、成员、权限、评论和通知。
- 独立电子表格、数据表、画板或演示文稿内容类型。
- 在线模板市场和公共内容发现；R001 只实现内置的离线模板选择器。
- 附件云上传、外链 CDN 或跨设备下载。
- 对语雀品牌、图标和专有视觉资源的复制。

这些能力如需实施，应分别建立后续需求，不作为 R001 完成的阻塞项。

## 4. 用户故事

### 4.1 知识库组织

- 作为每天使用应用的用户，我希望启动后直接看到最近编辑和浏览的文档，而不是先选择知识库。
- 作为需要快速记录的用户，我希望从首页直接新建文档，并选择目标知识库和分组。
- 作为重复创建相似文档的用户，我希望从内置模板生成周报、会议纪要或技术笔记。
- 作为个人用户，我希望为知识库设置图标和简介，并有一个首页快速了解知识库内容。
- 作为拥有大量文档的用户，我希望用分组组织文档，并在分组之间拖动、排序和折叠文档。
- 作为高频写作者，我希望快速找到最近编辑、最近浏览或收藏的文档。

### 4.2 编辑

- 作为不熟悉 Markdown 的用户，我希望通过顶部工具栏完成常见排版。
- 作为技术写作者，我希望设置代码块语言并获得语法高亮。
- 作为长文作者，我希望随时看到保存状态和字数，并能恢复过去的本地版本。
- 作为资料整理者，我希望插入文件附件并能在离线状态下打开或下载。

## 5. 信息架构与页面结构

### 5.1 首页层级与路由

```text
应用
├── 全局侧栏
│   ├── 搜索
│   ├── 开始
│   ├── 最近
│   ├── 收藏
│   ├── 回收站
│   ├── 设置
│   └── 知识库列表
└── 主区域
    ├── 开始/工作台首页
    ├── 最近、收藏、搜索、回收站等全局视图
    └── 当前知识库
        ├── 知识库头部：图标、名称、简介、菜单
        ├── 知识库首页
        ├── 目录：分组与文档
        └── 文档编辑区
```

首次安装进入“开始”首页；后续启动恢复上次路由。全局侧栏的“开始”始终返回工作台首页，不等同于知识库内部的“首页”。

### 5.2 全局“开始/工作台”首页

#### 5.2.1 页面结构

```text
开始
├── 快速操作卡片
│   ├── 新建文档
│   ├── 新建知识库
│   ├── 模板中心
│   └── AI 帮你写
└── 文档活动区
    ├── 状态页签：编辑过 / 浏览过
    ├── 筛选：类型 / 归属知识库
    └── 文档表格：标题 / 类型 / 归属 / 时间
```

本地单人模式不显示语雀中的“提到我、我点赞的、我评论过、邀我协作、分享中的、创建者”控件。这些筛选依赖账号或协作数据，留给后续协作需求。

#### 5.2.2 快速操作卡片

**新建文档**

- 点击卡片主区域时，优先在最近使用的知识库根目录创建“无标题”文档并进入编辑区。
- 卡片提供下拉入口，可先选择目标知识库和分组；没有最近知识库时必须先选择。
- 创建后聚焦标题输入框，正文保持空白。
- R001 只提供“文档”类型，不展示不可用的表格、画板和数据表入口。

**新建知识库**

- 打开轻量创建弹窗，必填名称，可选图标和描述。
- 创建成功后进入该知识库首页，不自动创建文档。

**模板中心**

- 使用本地内置模板，不访问网络。
- 首批模板：空白文档、会议纪要、周报、项目计划、技术方案、读书笔记。
- 模板卡片显示名称、用途和内容预览；用户选择目标知识库/分组后创建副本。
- 模板正文以静态 Tiptap JSON 随应用发布，创建后与普通文档无差别。

**AI 帮你写**

- 未配置 AI 时打开现有设置面板并说明需要配置兼容服务。
- 已配置时打开“新建 AI 文档”流程：选择知识库/分组、输入主题、选择文档类型、生成预览、确认创建。
- AI 生成结果必须在确认后才写入新文档；取消流程不创建空文档。
- 文档类型提示词首批提供：自由写作、方案、总结、周报、会议纪要。

#### 5.2.3 文档活动区

- 默认页签为“编辑过”，按 `updatedAt` 倒序。
- “浏览过”按 `lastOpenedAt` 倒序，没有浏览时间的文档不展示。
- 列表跨越所有知识库，但排除回收站和分组。
- 每行显示文档图标、标题、类型、归属路径和相对时间；归属格式为“知识库 / 分组路径”。
- 点击文档标题打开文档；点击归属路径进入对应知识库目录并定位该文档。
- 类型筛选在 R001 只有“文档”，保留接口但不显示只有一个选项的下拉。
- 归属筛选支持全部知识库或单个知识库。
- 首次加载 30 条，点击“查看更多”每次增加 30 条，不使用无限滚动。
- 空状态区分“尚未编辑文档”“尚未浏览文档”和“筛选后无结果”。

#### 5.2.4 “开始”导航预览

- 指针悬停或键盘聚焦全局侧栏“开始”入口时，可展开最近文档预览。
- 预览提供“编辑过/浏览过”切换，最多显示 5 条标题和相对时间。
- 预览只用于快速打开，不承载筛选和创建功能；移开焦点或按 Escape 后关闭。

### 5.3 知识库首页

知识库首页用于展示单个知识库的整体信息和目录，而不是重复全局最近列表。

首页至少包含：

- 知识库图标、名称、描述和收藏切换。
- 文档数量与正文总字数；分组不计入文档数量。
- “新建文档”和“新建分组”主操作。
- 欢迎说明：优先显示知识库描述；描述为空时显示知识库用途引导。
- 完整目录概览，按当前目录排序展示顶层分组和文档。
- 分组在首页作为分段标题；文档显示标题和最后更新时间。
- 首页目录允许展开/收起分组，但不在首页执行拖拽；结构调整统一在“目录”视图完成。

空知识库首页显示引导，不自动创建无标题文档。R001 没有远程分享能力，因此不得放置无功能的分享占位控件。

### 5.4 目录与分组

- 目录根节点允许同时存在分组和文档。
- 分组可以嵌套分组或文档，不限制业务层级；界面默认最多展开到当前可见深度。
- 创建分组时默认名称为“新建分组”，立即进入重命名状态。
- 创建文档时默认名称为“无标题”，立即选中并聚焦标题。
- 拖动支持目标前、目标内部、目标后三种落点，并继续禁止循环父级。
- 分组删除时整棵子树进入回收站；恢复后回到原父级和尽可能接近原排序位置。
- 分组本身不承载富文本正文。
- 当前已有“文件夹”在界面统一显示为“分组”。

### 5.5 最近与收藏

- 独立“最近”视图复用开始首页活动区的数据和列表组件，包含“最近编辑”和“最近浏览”两个页签。
- 最近编辑使用文档 `updatedAt`，最近浏览使用 `lastOpenedAt`；都跨知识库且只展示未删除文档。
- 用户打开文档并在主区域完成渲染后更新 `lastOpenedAt`，仅预览搜索结果不更新。
- 收藏同时支持知识库和文档；全局收藏页先显示收藏知识库，再显示收藏文档。
- 编辑区标题栏、知识库首页和列表项均提供收藏切换。
- 删除文档不清除收藏时间，恢复后仍保持收藏；永久删除时一并删除元数据。
- 收藏知识库不会改变其在全局侧栏中的手动排序，只在收藏页和快速入口中优先展示。

## 6. 数据模型与迁移

### 6.1 领域模型调整

```ts
type PageKind = "document" | "group";

interface Workspace {
  id: string;
  name: string;
  icon: string | null;
  description: string;
  homePageId: string | null;
  favoriteAt: number | null;
  lastOpenedAt: number | null;
  createdAt: number;
  updatedAt: number;
}

interface Page {
  id: string;
  workspaceId: string;
  parentId: string | null;
  kind: PageKind;
  title: string;
  icon: string | null;
  position: number;
  favoriteAt: number | null;
  lastOpenedAt: number | null;
  deletedAt: number | null;
  createdAt: number;
  updatedAt: number;
}

interface DocumentRevision {
  id: string;
  pageId: string;
  contentJson: unknown;
  textSnapshot: string;
  createdAt: number;
  reason: "interval" | "before-restore" | "manual";
}

interface Attachment {
  id: string;
  pageId: string;
  name: string;
  mimeType: string;
  size: number;
  blob: Blob;
  createdAt: number;
}
```

`homePageId` 为后续自定义首页预留；R001 默认使用系统生成的知识库首页，不要求创建正文首页文档。

### 6.2 IndexedDB 升级

- 数据库版本从当前版本升级一版，新增 `revisions` 和 `attachments` store。
- 为 `Page` 补齐 `favoriteAt`、`lastOpenedAt`，默认值为 `null`。
- 为 `Workspace` 补齐 `icon`、`description`、`homePageId`、`favoriteAt` 和 `lastOpenedAt`，时间字段默认值为 `null`。
- 将旧 `Page.kind === "folder"` 原地迁移为 `group`，保留 `id`、父级和排序。
- `revisions` 对 `pageId` 和 `[pageId, createdAt]` 建索引；`attachments` 对 `pageId` 建索引。
- 迁移必须在事务中完成；任一记录失败时不得写入半完成的新结构。
- 升级后重复打开数据库不应重复迁移或重复创建数据。

### 6.3 兼容与回滚原则

- 迁移前后的文档 JSON 不做格式转换。
- 所有读取函数继续对字段缺失和非法形状降级，旧备份导入时可补默认值。
- 迁移测试必须使用真实旧版本 fixture，验证层级、内容、标签、回收站和偏好均保留。

## 7. 编辑区需求

### 7.1 编辑区布局

```text
文档顶栏
├── 面包屑/文档标题
├── 收藏
├── 保存状态
├── 字数
├── 版本历史
├── Markdown 导出
├── 目录
└── 主题

常驻格式工具栏
├── 插入
├── 撤销/重做
├── 段落样式
├── 字号
├── 行内格式
├── 颜色/高亮
├── 对齐
├── 列表/缩进
├── 任务
└── 更多

文档画布
└── 现有块把手、选区工具栏、表格工具条和 AI 面板
```

桌面端常驻显示完整工具栏；宽度不足 1024px 时将低频命令收进“更多”；小于 768px 时工具栏允许横向滚动，不压缩编辑正文。

### 7.2 常驻工具栏命令

必做命令如下：

| 分组 | 命令 |
| --- | --- |
| 历史 | 撤销、重做 |
| 段落 | 正文、标题 1–6、引用、代码块 |
| 字号 | 默认、12、14、16、18、20、24、28、32px |
| 行内 | 加粗、斜体、下划线、删除线、行内代码、上标、下标 |
| 链接 | 新建/编辑/移除链接 |
| 颜色 | 文本颜色、高亮背景、恢复默认 |
| 段落布局 | 左对齐、居中、右对齐、两端对齐 |
| 列表 | 项目列表、编号列表、任务列表 |
| 层级 | 增加缩进、减少缩进 |
| 插入 | 分隔线、图片、附件、表格、行内公式、公式块、Emoji、AI 助手 |
| 清理 | 清除行内格式、将当前块重置为正文 |

### 7.3 命令统一

- 常驻工具栏、`/` 菜单、选区工具栏和块菜单必须复用同一命令定义或同一命令执行函数。
- 命令定义包含 `id`、名称、分组、图标、快捷键、可用条件、激活条件和执行逻辑。
- 不适用于当前选区的命令显示为禁用，不静默失败。
- 执行命令后焦点返回编辑器；打开弹层类命令除外。

### 7.4 标题、字号与缩进

- Tiptap schema 支持标题 1–6；现有标题 1–3 内容保持不变。
- 字号作为 `textStyle` 属性保存；“默认”移除字号属性，使用主题正文大小。
- 增减缩进仅作用于列表项或支持缩进的文本块，最大 8 级。
- `Tab`/`Shift+Tab` 在列表中执行缩进，在非列表文本中不插入不可控的制表符。

### 7.5 代码块

- 代码块提供语言选择：纯文本、JavaScript、TypeScript、JSON、HTML、CSS、Shell、Python、C、C++、Rust、Java。
- 语言保存在代码块属性中；未知语言回退为纯文本，不破坏正文。
- 代码内容可复制；复制按钮只写剪贴板，不改变文档。
- 语法高亮在浅色、深色主题下均满足可读性。

### 7.6 附件块

- 单附件上限 20MB；超过限制时在选择后立即提示，且不写入 IndexedDB。
- 附件以 Blob 存入 `attachments` store，文档节点只保存附件 ID、名称、类型和大小。
- 附件块显示文件名、类型、大小、下载和移除操作。
- 下载使用本地 Blob URL；生成后及时释放 URL。
- 删除附件块仅删除文档引用；保存完成后执行孤儿附件清理。
- 永久删除文档时级联删除其附件；进入回收站时保留附件以便恢复。
- 图片仍使用现有图片节点；图片插入不自动改为附件块。

## 8. 文档状态与本地版本历史

### 8.1 保存状态

状态机定义为：

```text
已保存 ──编辑──> 有未保存更改 ──防抖开始──> 保存中
保存中 ──成功──> 已保存
保存中 ──失败──> 保存失败 ──重试──> 保存中
```

- 编辑后立即显示“有未保存更改”。
- 保存开始显示“保存中…”，成功后显示“已保存 HH:mm:ss”。
- 保存失败显示明确错误和“重试”按钮，不清除编辑器中的当前内容。
- 切换文档和卸载前继续执行现有 flush；若同步 flush 无法完成，保留内存内容直至页面生命周期结束，不伪造成功。

### 8.2 字数统计

- 中文、日文、韩文单个字符计 1；连续拉丁字母或数字计 1 个词；标点和空白不计。
- 工具栏显示紧凑值，例如“1,528 字词”；点击后显示字符数、词数和段落数。
- 统计来自当前编辑器文档，不依赖已保存的 `textSnapshot`，避免显示滞后。

### 8.3 本地版本历史

- 自动保存仍保持当前防抖频率，不为每次保存生成版本。
- 文档发生变化且距离上一个自动版本至少 5 分钟时，保存成功后创建 `interval` 版本。
- 每篇文档最多保留 100 个自动版本；手动版本和恢复前版本不受自动清理影响。
- 版本列表显示时间、原因和正文摘要，不渲染完整可编辑实例。
- 恢复版本前先将当前内容保存为 `before-restore` 版本，然后将选中版本写入当前文档。
- 相邻版本内容完全一致时不重复创建，可通过稳定内容哈希判断。
- 永久删除文档时级联删除所有版本；进入回收站时保留。

## 10. 异常与边界情况

- 删除当前正在编辑的文档后，主区域返回知识库首页，不保留失效编辑器实例。
- 分组或文档重名允许存在，以层级和 ID 区分。
- 全局最近或收藏视图没有文档时显示对应引导空态，不回退显示已删除内容。
- 不存在任何知识库时，开始首页禁用“直接新建文档”和“AI 帮你写”，并突出“新建知识库”；模板中心允许浏览，但创建时要求先建知识库。
- 最近使用的目标知识库已被永久删除时，新建文档流程回退到知识库选择，不复用失效 ID。
- 附件 Blob 读取失败时显示“附件不可用”，允许移除损坏节点。
- 版本记录损坏时跳过该记录，其他版本仍可查看。
- 浏览器存储空间不足时，保存或附件写入进入“保存失败”状态并提示释放空间；不得删除旧数据作为自动补救。
- 数据迁移完成前显示加载状态；迁移失败时阻止写入并显示恢复建议。

## 14. 风险与控制

| 风险 | 控制方式 |
| --- | --- |
| `folder` → `group` 破坏旧数据 | 使用真实 v1 fixture 迁移测试；不改变 ID 和父子关系 |
| 常驻工具栏与现有菜单逻辑分叉 | 扩展统一命令注册表，所有入口调用同一执行层 |
| 版本历史快速膨胀 | 5 分钟间隔、内容去重、自动版本每文档上限 100 |
| 附件占满浏览器存储 | 20MB 单文件限制、写入失败提示、孤儿清理，不自动删除用户数据 |
| 工具栏挤压窄屏 | 低频项折叠、移动端横向滚动、增加断点视觉测试 |
| 大文档统计或高亮卡顿 | 字数随编辑事务节流计算；代码高亮仅处理代码块 |
