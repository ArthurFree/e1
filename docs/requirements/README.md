# 增量需求目录

本目录用于记录基于产品对比、用户反馈和实现复盘产生的增量需求。根目录的 [`../requirements.md`](../requirements.md) 描述产品长期基线；本目录中的文档描述可独立设计、实现和验收的具体迭代。

## 编号与更新规则

- 文件名使用 `Rxxx-主题.md`，编号一经分配不复用。
- 同一需求的澄清、方案调整、阶段进度和验收结果持续更新原文件，保持链接稳定。
- 每份需求必须记录版本号、状态、更新时间和变更记录。
- 状态依次为：`草案`、`已确认`、`实现中`、`待验收`、`已完成`；暂不实施时标记为 `已延期`。
- 开始实现前，需求必须明确范围、数据模型、交互规则、迁移方案、阶段拆分和验收标准。
- 实现中每完成一个阶段，在对应阶段清单中更新状态并补充实际偏差；不得只修改代码而不回写需求。
- 会改变基础架构或产品边界的决定，同时更新 `../architecture.md`、`../decisions.md` 和相关测试文档。

## 单份需求的完成定义

一份需求只有同时满足以下条件才可标记为“已完成”：

1. 文档中所有必做验收项通过，延期项有明确后续编号。
2. 数据迁移可重复执行，已有本地数据不会丢失。
3. 单元、组件、端到端和视觉回归测试通过。
4. 产品文档、架构文档、决策记录和 `AGENTS.md` 已同步真实状态。
5. 已记录实现中的取舍、遗留问题与下一轮入口。
