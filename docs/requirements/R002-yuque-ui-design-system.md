## 1. 背景

R001 已补齐语雀式知识库结构、全局开始首页、知识库首页、常驻工具栏、最近与收藏等核心能力，但现有界面仍沿用早期 Notion-like 视觉：最左侧为 56px 图标轨道、颜色偏冷蓝、页面与组件密度不统一，新增功能虽然可用，却没有形成一致的产品外观。

本需求在不改变 R001 数据模型和业务规则的前提下，建立一套适合中文知识库产品的 UI 系统，并重构三个核心场景：

1. 全局开始/工作台；
2. 知识库首页与目录；
3. 文档编辑区。

设计参考来自 2026-07-22 对本机语雀客户端和已登录语雀网页工作台的只读观察。参考对象用于提炼布局、密度、层级和交互规律，不复制语雀 Logo、插画、专有图标、文案资产或其他品牌资源。

## 2. 调研结论

### 2.1 本机客户端观察

#### 全局工作台

- 左侧是完整的全局导航，不是只有图标的窄轨道；顶部包含账户、搜索和新建入口。
- 主导航与知识库列表处于同一侧栏，通过分组标题区分层级。
- 主区顶部以四张横向快捷卡片承载“新建文档、知识库、模板、AI”高频任务。
- 最近文档采用三列轻量表格：名称、归属路径、操作时间；行间仅使用低对比分隔线。
- 页签和筛选器在同一行；筛选器弱化，内容列表才是视觉主体。
- 导航悬停会打开最近内容预览浮层，浮层为白色、轻阴影、约 8px 圆角。

#### 知识库与目录

- 进入知识库后出现第二层侧栏；左侧全局导航仍保留。
- 知识库侧栏顶部显示图标、名称、返回入口和新增按钮。
- “首页”“目录”位于树结构之前，形成清晰的固定入口。
- 分组标题、文档标题和层级缩进均保持 14px 紧凑行高；选中项使用整行浅灰底。
- 知识库首页以标题、简介、统计、操作按钮和完整目录概览为主，不使用大型营销卡片。

#### 编辑器

- 文档状态栏与格式工具栏分为两行；两者均固定在主区顶部。
- 文档标题、保存状态、收藏、协作和分享操作在第一行；格式命令在第二行。
- 工具栏按钮主要为 28–32px 点击区域，图标约 16px，分组之间以细竖线或留白分隔。
- 正文列约 760–840px，左右保留大面积白色空间，正文首屏顶部留白明显。
- 标题为约 30–32px 中等字重；正文为 16px 左右，行高宽松。
- 大纲、字数和辅助操作使用低强调度，不与正文争夺注意力。

### 2.2 网页端核对

在 1622×1080 桌面视口下，网页端的可观测基础样式为：

- 全局正文：14px；
- 主文本色：`rgb(38, 38, 38)`，即 `#262626`；
- 字体栈以系统字体、`PingFang SC`、`Microsoft YaHei` 为主；
- 左侧全局侧栏约 253px；
- 导航行高约 32px；
- 快捷卡片标题为 14px / 500；
- 页面背景以纯白和极浅灰分区，阴影仅用于浮层。

在 1024×768 视口下，网页端仍保留约 253px 全局侧栏，主列表缩小并截断标题。项目不完全照搬这一行为：1024px 以下应优先折叠侧栏，以保证编辑器和表格的可用宽度。

### 2.3 可复用的设计原则

1. **层级来自布局，不来自装饰**：用侧栏、留白、字号和细边框区分区域，避免大面积阴影与高饱和色块。
2. **中文优先的紧凑密度**：导航、列表和工具栏默认 14px，单行高度 32–40px。
3. **内容优先**：正文和文档列表保持白底；侧栏以极浅灰承载导航。
4. **操作渐进显现**：低频行内操作默认隐藏，在悬停、选中或键盘聚焦时显示。
5. **状态始终可感知**：选中、保存、拖拽、禁用、错误和离线状态不能只依赖颜色。
6. **组件一致**：相同语义的按钮、菜单、弹层、空态和加载态在所有页面复用同一规格。

## 3. 目标与非目标

### 3.1 目标

- 建立可直接映射为 CSS 变量的颜色、排版、间距、圆角、阴影、层级和动效令牌。
- 将当前窄轨道改造成桌面端完整全局侧栏，形成“全局侧栏—知识库侧栏—主区”三级布局。
- 统一开始首页、知识库首页和编辑器的视觉语言与交互状态。
- 保证 1440、1280、1024、768、390 五类宽度下均可使用。
- 浅色和深色主题同时落地，不允许只完成浅色后再机械反色。
- 建立视觉回归基线和组件状态页，降低后续迭代的视觉漂移。

### 3.2 非目标

- 不调整 R001 的领域模型、路由语义、数据存储和迁移逻辑。
- 不新增实时协作、远程分享、评论后端和多人头像数据。
- 不复刻语雀 Logo、品牌绿色、插画和专有图标；项目继续使用自有名称和中性图形资产。
- 不在本需求中更换编辑器框架或重写 Tiptap 扩展。
- 不为了像素一致而引入运行时 CSS-in-JS 或新的大型 UI 框架。

## 4. 信息架构与页面骨架

### 4.1 桌面应用壳

```text
┌──────────────┬──────────────────┬─────────────────────────────────────┐
│ 全局侧栏 240 │ 知识库侧栏 224   │ 主区                                │
│              │                  │ ┌ 文档/页面顶栏 48 ───────────────┐ │
│ 账户与搜索   │ 知识库名称       │ ├ 格式工具栏 42（编辑页）────────┤ │
│ 开始/AI/小记 │ 首页/目录        │ │                                   │ │
│ 收藏/最近    │ 分组与文档树     │ │ 页面内容 / 编辑画布               │ │
│ 知识库列表   │                  │ │                                   │ │
│ 设置/回收站  │                  │ │                                   │ │
└──────────────┴──────────────────┴─────────────────────────────────────┘
```

布局规则：

- 全局侧栏默认宽度 240px，允许用户收窄到 208px，最大 280px。
- 知识库侧栏默认 224px，可拖动范围 200–320px；宽度沿用现有偏好持久化。
- 主区最小可用宽度为 560px；不足时按响应式规则折叠侧栏，禁止继续压缩正文。
- 页面顶栏高 48px；编辑工具栏高 42px；两行之间用 `1px` 细边框分隔。
- 三栏均独立滚动。顶栏和工具栏固定，正文滚动不得带走顶栏。
- 全局侧栏与知识库侧栏使用不同但接近的浅灰层级，主区保持纯白。

### 4.2 开始首页骨架

```text
┌ 全局侧栏 ────┬──────────────────────────────────────────────────────┐
│               │ 开始                                                │
│               │                                                     │
│               │ [新建文档⌄] [新建知识库] [模板中心] [AI 帮你写]    │
│               │                                                     │
│               │ 文档                                                │
│               │ [编辑过][浏览过][我点赞的][我评论过]  类型 归属…  │
│               │ ──────────────────────────────────────────────────  │
│               │ ▣ 标题                     归属路径       时间      │
│               │ ▣ 标题                     归属路径       时间      │
└───────────────┴─────────────────────────────────────────────────────┘
```

- 内容容器最大宽度 1180px，左右边距桌面为 40px，1280px 以下为 24px。
- 页面标题 20px / 600，顶部间距 28px。
- 快捷卡片采用四列等宽布局，卡片高度 64px，间距 12px。
- 最近区域与快捷卡片之间间距 32px。
- 文档行高度 54px，名称列优先分配宽度；归属和时间列分别建议为 260px 与 140px。

### 4.3 知识库首页骨架

```text
┌ 全局侧栏 ┬ 知识库侧栏 ┬────────────────────────────────────────────┐
│          │ 首页        │ [图标] 知识库名称            收藏  ···   │
│          │ 目录        │        简介                                │
│          │             │        202 文档 · 151749 字                 │
│          │ 分组        │                                            │
│          │  文档       │ 目录                       新建文档 新建分组│
│          │  文档       │ 分组标题                                   │
│          │             │   ▣ 文档标题                    更新时间   │
└──────────┴─────────────┴────────────────────────────────────────────┘
```

- 首页内容最大宽度 960px，顶部 48px，左右最少 32px。
- 知识库图标为 40×40px；名称 24px / 600；简介 14px / 22px。
- 统计信息为 12px 次要文本，使用文本分隔符，不使用彩色徽章。
- 目录分组标题高 36px，文档行高 44px。
- 空知识库在内容列左对齐显示，不使用全屏居中插画。

### 4.4 编辑器骨架

```text
┌ 全局侧栏 ┬ 目录侧栏 ┬ 文档标题  已保存          ☆  历史  分享  ··· ┐
│          │          ├ ＋ ↶ ↷  段落  16px  B I U 颜色 列表 对齐… ┤
│          │          │                                             │
│          │          │        文档标题                             │
│          │          │                                             │
│          │          │        正文内容（760–840px）                │
│          │          │                                             │
│          │          │                         大纲 / 字数          │
└──────────┴──────────┴─────────────────────────────────────────────┘
```

- 顶栏左侧标题超过 320px 截断，完整内容由 Tooltip 展示。
- 保存状态紧跟标题，字号 12px；正常为次要色，保存中显示旋转图标，失败显示错误色与重试。
- 顶栏右侧文本按钮最多保留一个主要操作，其他均为图标按钮。
- 工具栏按“插入、历史、块类型、文字、段落、更多”分组。
- 正文默认宽度 780px，可在设置中切换 780px 与 920px 两档；移动端为 `100%`。
- 编辑标题 32px / 1.25 / 600，标题与正文间距 24px。
- 正文 16px / 1.75，段落间距 8px；中文长文阅读每行约 35–45 个汉字。

## 5. 设计令牌

### 5.1 颜色

所有业务组件只能使用语义变量，不直接书写十六进制颜色。

| 变量 | 浅色 | 深色 | 用途 |
| --- | --- | --- | --- |
| `--color-canvas` | `#FFFFFF` | `#171817` | 主画布 |
| `--color-sidebar-global` | `#F7F8F7` | `#1D1F1E` | 全局侧栏 |
| `--color-sidebar-context` | `#FBFBFA` | `#202220` | 知识库侧栏 |
| `--color-surface` | `#FFFFFF` | `#252725` | 卡片、菜单、弹窗 |
| `--color-surface-hover` | `#F3F4F3` | `#2D302E` | 悬停 |
| `--color-surface-active` | `#EDEFEF` | `#353936` | 选中、按下 |
| `--color-text-primary` | `#262626` | `#E8EAE8` | 标题与正文 |
| `--color-text-secondary` | `#737773` | `#A6AAA6` | 描述与元信息 |
| `--color-text-tertiary` | `#A0A4A0` | `#777C78` | 占位、禁用 |
| `--color-border-subtle` | `#EDEEEB` | `#303330` | 区域与行分隔 |
| `--color-border-strong` | `#D8DBD8` | `#474B48` | 输入框、弹窗边界 |
| `--color-brand` | `#22A06B` | `#42C98A` | 项目主色、焦点、主按钮 |
| `--color-brand-soft` | `#E8F7F0` | `#173D2D` | 品牌弱背景 |
| `--color-info` | `#3578E5` | `#6DA0F2` | 信息、文档图标 |
| `--color-warning` | `#D9822B` | `#F0A454` | 警告 |
| `--color-danger` | `#D14343` | `#F06B6B` | 删除、保存失败 |
| `--color-overlay` | `rgba(18,20,18,.42)` | `rgba(0,0,0,.64)` | 模态遮罩 |

补充规则：

- 知识库默认图标使用 `--color-info`，但允许用户图标颜色覆盖。
- 活动列表图标、选中背景和主操作不得同时使用高饱和品牌色。
- 深色主题不是浅色的反转；边框、悬停和浮层需单独校准。

### 5.2 字体与字号

字体栈：

```css
"Chinese Quote", -apple-system, BlinkMacSystemFont, "Segoe UI",
Roboto, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
"Helvetica Neue", Arial, sans-serif
```

| 令牌 | 字号/行高 | 字重 | 场景 |
| --- | --- | --- | --- |
| `caption` | 12/18px | 400 | 时间、状态、辅助说明 |
| `body-sm` | 13/20px | 400 | 菜单说明、紧凑元信息 |
| `body` | 14/22px | 400 | 导航、表格、控件 |
| `body-medium` | 14/22px | 500 | 卡片标题、选中导航 |
| `editor-body` | 16/28px | 400 | 正文 |
| `section-title` | 16/24px | 600 | 页面分区标题 |
| `page-title` | 20/28px | 600 | 开始首页标题 |
| `knowledge-title` | 24/32px | 600 | 知识库名称 |
| `document-title` | 32/40px | 600 | 文档标题 |

- 中文界面避免 300 细字重；Windows 上 500 可回退为 600。
- 单行控件内文本垂直居中；不可用 `line-height: normal` 依赖浏览器差异。
- 代码字体统一使用 `SFMono-Regular, Consolas, Liberation Mono, monospace`。

### 5.3 间距、尺寸与圆角

- 基础间距序列：`4, 8, 12, 16, 20, 24, 32, 40, 48px`。
- 图标：默认 16px，小图标 14px，大图标 20px。
- 点击区域：图标按钮最小 32×32px；移动端最小 40×40px。
- 控件高度：紧凑 28px、默认 32px、输入框 36px、顶栏 48px。
- 圆角：导航项 6px、按钮/输入框 6px、卡片 8px、浮层 10px、模态框 12px。
- 边框：默认 1px；选中不得通过增加边框宽度造成布局抖动。

### 5.4 阴影与层级

```css
--shadow-popover: 0 6px 20px rgba(18, 20, 18, 0.12),
                  0 1px 4px rgba(18, 20, 18, 0.08);
--shadow-dialog: 0 18px 48px rgba(18, 20, 18, 0.20),
                 0 2px 8px rgba(18, 20, 18, 0.10);
```

| 层级 | z-index | 内容 |
| --- | --- | --- |
| Base | 0 | 页面与侧栏 |
| Sticky | 10 | 顶栏、工具栏 |
| Dropdown | 30 | 下拉菜单、Tooltip |
| Popover | 40 | 搜索、悬停预览、颜色面板 |
| Drawer | 50 | 响应式侧栏 |
| Modal | 60 | 对话框与遮罩 |
| Toast | 70 | 全局反馈 |

页面主体、静态卡片禁止使用阴影。阴影只用于浮出当前平面的内容。

### 5.5 动效

- 快速反馈 120ms：悬停色、按钮按下。
- 标准反馈 180ms：菜单、Tooltip、内容折叠。
- 结构变化 240ms：抽屉和侧栏展开。
- 缓动统一为 `cubic-bezier(.2, .8, .2, 1)`。
- 尊重 `prefers-reduced-motion`；开启后取消位移和缩放，只保留不超过 80ms 的透明度变化。
- 列表重排和拖拽不得对整页添加过渡。

## 6. 图标与资产规范

- 核心导航和工具栏统一使用同一套线性 SVG 图标，建议延续项目现有 Lucide 风格。
- 默认为 16px、1.75px 描边、圆角端点；同一行不得混用实心与线性风格。
- Emoji 仅允许作为用户自定义知识库/文档图标，不作为“开始、搜索、设置、回收站”等系统图标。
- 系统图标必须继承 `currentColor`，不得在 SVG 内硬编码颜色。
- 有文字标签的按钮不重复添加无意义装饰图标。
- 不引入语雀 Logo、吉祥物、插画和专有图标文件。

## 7. 核心组件规格

### 7.1 全局侧栏 `GlobalSidebar`

- 宽 240px，高度占满视口，背景 `--color-sidebar-global`。
- 顶部账户行高 44px；搜索行高 36px；导航项高 32px。
- 一级导航图标与文字间距 10px，左右内边距 12px。
- 选中项为浅灰整行背景 + 500 字重；不增加左侧彩条。
- 知识库列表与一级导航之间留 12px，并显示可折叠分组标题。
- 列表项悬停时显示锁、更多操作；无悬停时只保留必要状态。
- 底部“设置、回收站、更多”固定，不随知识库列表滚动。
- 折叠态宽 64px，仅用于 1024–1279px；折叠后图标显示 Tooltip。

### 7.2 搜索入口 `SearchTrigger`

- 高 32px，浅灰背景，圆角 6px；左侧搜索图标，右侧快捷键。
- 默认不显示实线边框，悬停提升背景对比；键盘聚焦显示 2px 品牌色焦点环。
- 点击打开全局搜索弹窗；入口本身不得伪装成可编辑输入框。

### 7.3 快捷创建卡片 `QuickActionCard`

- 最小宽度 180px，高 64px，1px 细边框，8px 圆角，无阴影。
- 左侧图标 32×32px 弱色背景；右侧标题 14px / 500，描述 12px。
- 悬停：边框变强、背景轻微变灰；禁止上浮和大阴影。
- 若含下拉，右侧 32px 区域独立可点击，并有分隔线和明确 `aria-label`。
- 禁用时保留布局，降低对比并显示原因 Tooltip。

### 7.4 页签与筛选

- 活动页签使用文字型分段控件，高 32px。
- 选中项为浅灰圆角底 + 500 字重；不得只用绿色文字。
- 筛选器为 32px 文本按钮，标签与下箭头间距 4px。
- 1280px 以下将低频页签收进“更多”；筛选器保持可见。

### 7.5 活动列表 `ActivityList`

- 表头可省略；列含义通过内容结构和筛选栏明确。
- 行高 54px，底部 1px 分隔线；悬停背景仅覆盖行。
- 标题 14px / 500，归属和时间 13px / 次要色。
- 名称列允许一行截断；归属路径中间截断，保留知识库名和末级分组。
- 行内收藏和更多按钮只在悬停、焦点或已收藏时显示。
- 加载使用与实际列宽一致的骨架行；空态保留页签和筛选器。

### 7.6 知识库侧栏与树

- 顶部知识库行高 48px；图标 24px；新增按钮 28px。
- “首页”“目录”固定项高 32px，树区与固定项间有 8px 间距。
- 分组行和文档行高 30px；每级缩进 16px，最深可视层级为 6。
- 展开箭头 14px，点击区 24px；箭头点击仅展开，标题点击仅选择。
- 文档选中项使用整行浅灰底，背景需覆盖缩进区。
- 拖拽时显示 2px 品牌色插入线；作为子项时高亮目标行并显示缩进预览。
- 重命名输入框高 28px，原位替换文本，不打开额外弹窗。

### 7.7 页面与编辑器顶栏

- 高 48px，左右内边距 12px，底部 1px 分隔线。
- 左侧：折叠侧栏按钮、文档标题、保存状态。
- 右侧：收藏、历史、主题或更多、主要操作；图标按钮 32px。
- 操作超过可用宽度时按优先级进入更多菜单，标题区域最少保留 160px。
- 不具备远程分享能力时不得出现可误解的“分享”按钮；可显示“导出”或“复制链接（本地）”。

### 7.8 格式工具栏

- 高 42px，横向排列，背景跟随主画布，底部细边框。
- 图标按钮 30×30px；下拉控件高 30px；组间距 4px，组间分隔 1px。
- 激活态：浅品牌色背景 + 主文本色图标；禁用态不响应悬停。
- 工具栏只显示图标时必须有 Tooltip，并标明快捷键。
- 低于 900px 可用宽度时，低频命令进入“更多”；移动端允许横向滚动，但标题栏不得横向滚动。
- 工具栏弹层以按钮边缘对齐，自动翻转，不能超出视口。

### 7.9 按钮、输入框和选择器

- 主要按钮：品牌色底、白字，默认高 32px；同一视图最多一个主要按钮组。
- 次要按钮：白底细边框；幽灵按钮：透明底；危险操作仅在确认阶段使用红色。
- 输入框默认高 36px，左右 10px；错误态同时提供错误文案和图标。
- 下拉菜单最小宽 180px，单项高 32px，危险项与普通项用分隔线隔开。

### 7.10 弹层、对话框、抽屉与反馈

- Tooltip 延迟 400ms 出现，离开 80ms 消失；只承载短说明和快捷键。
- Popover 使用 10px 圆角和 `--shadow-popover`，内边距 6–12px。
- Modal 默认宽 440px，最大宽度 `calc(100vw - 32px)`；标题 16px / 600。
- Drawer 从左侧进入，用于 1024px 以下的侧栏；宽度为 `min(320px, 86vw)`。
- Toast 位于右下角，移动端位于底部居中；成功 3 秒消失，错误需手动关闭或 8 秒后消失。
- 删除、恢复版本、清空回收站保留二次确认；普通导航和样式切换不得弹确认框。

### 7.11 空态、加载态和错误态

- 空态由标题、单句说明和一个主要操作组成；插画可选，不作为依赖。
- 页面首次加载使用结构化骨架，不显示全屏旋转器。
- 局部保存使用顶栏状态；全局不可用才显示页面级错误。
- 离线时顶栏显示“离线，内容将保存在本机”，恢复后自动变为“已保存”。
- 所有错误提供下一步：重试、返回或打开设置，不能只显示错误码。

## 8. 交互状态矩阵

所有可交互组件至少实现以下状态：

| 状态 | 视觉 | 行为 |
| --- | --- | --- |
| Default | 中性背景与文本 | 可点击 |
| Hover | 背景轻微增强 | 鼠标指针正确 |
| Active | 更深背景或 1px 内描边 | 即时反馈 |
| Selected | 整行/整块选中 + 字重变化 | 与焦点独立 |
| Focus-visible | 2px 品牌焦点环 | 键盘可见 |
| Disabled | 降低对比但保持可读 | 不触发动作，说明原因 |
| Loading | 保持原尺寸，显示进度 | 防止重复提交 |
| Error | 错误色 + 文案/图标 | 提供恢复入口 |

特别规则：

- Hover 不得覆盖 Selected；组合状态应表现为“选中背景 + 更强描边”。
- 拖拽时 Esc 取消，放置目标和原位置同时可辨识。
- 菜单打开后触发按钮保持 Active，菜单关闭后恢复。
- 鼠标和键盘触发的功能一致，不能把关键动作只放在右键菜单。

## 9. 响应式设计

### 9.1 `>= 1440px`

- 全局侧栏 240px + 知识库侧栏 224px + 主区。
- 快捷创建四列；活动列表显示名称、归属和时间。
- 编辑工具栏显示全部核心命令。

### 9.2 `1280–1439px`

- 全局侧栏 224px，知识库侧栏 216px。
- 开始首页边距缩至 24px。
- 工具栏低频命令可进入“更多”。

### 9.3 `1024–1279px`

- 全局侧栏默认折叠为 64px 图标栏，可临时展开为覆盖层。
- 知识库侧栏保持 216px；编辑正文仍不小于 620px 可视区。
- 活动列表时间列压缩，归属路径更早截断。

### 9.4 `768–1023px`

- 全局侧栏隐藏到抽屉；知识库侧栏可固定或覆盖，由 900px 断点决定。
- 开始首页快捷卡片改为两列。
- 活动列表隐藏归属列，将归属放到标题下第二行。
- 编辑工具栏横向滚动，右侧提供渐隐提示。

### 9.5 `< 768px`

- 单栏布局，两个侧栏均为抽屉且互斥打开。
- 顶栏高 48px，只保留菜单、保存状态和更多。
- 快捷卡片单列；活动列表为卡片式行，不呈现表格列。
- 正文左右边距：390px 视口为 20px，最小 16px。
- 文档标题 28px，正文仍为 16px。
- 浮动菜单和颜色面板最大宽度为 `calc(100vw - 24px)`。

## 10. 深色主题

- 深色主题的侧栏需与主画布保持可见层级，不能全部使用同一个黑色。
- 白色正文图片、代码块、引用块和表格必须分别检查边界。
- 阴影在深色主题中减弱扩散，主要依靠边框区分浮层。
- 品牌色用于焦点和主要按钮，不用于大面积背景。
- 系统跟随、浅色、深色三种选择沿用现有设置，并在首帧前应用，避免闪烁。

## 11. 可访问性

- 正常正文与背景对比度至少 4.5:1，大文本至少 3:1。
- 所有图标按钮提供可读 `aria-label`；Tooltip 不能替代无障碍名称。
- 导航、树、页签、菜单和对话框使用正确 ARIA 语义。
- 树支持上下键移动、左右键展开/收起、Enter 打开、F2 重命名。
- Escape 按层级关闭菜单、弹层、抽屉和对话框，不应直接离开编辑内容。
- 焦点不能被 Sticky 顶栏遮挡；打开 Modal/Drawer 后焦点锁定，关闭后归还触发器。
- 状态不得只依赖颜色；保存失败、选中、禁用、危险操作均需形状或文字辅助。
- 触摸场景的关键点击区不小于 40×40px。

## 12. CSS 与组件落地结构

### 12.1 样式目录

现有 `src/styles/global.css` 将按职责拆分，但继续使用原生 CSS，避免引入额外运行时：

```text
src/styles/
├── index.css                 # 统一导入顺序
├── tokens.css                # 浅色/深色语义令牌
├── reset.css                 # box-sizing、基础元素和焦点基础
├── typography.css            # 编辑正文与通用排版
├── layout.css                # 应用壳、三栏、响应式断点
├── utilities.css             # 少量可复用辅助类
└── components/
    ├── sidebar.css
    ├── toolbar.css
    ├── buttons.css
    ├── forms.css
    ├── menus.css
    ├── dialogs.css
    ├── lists.css
    └── editor.css
```

导入顺序固定为 `tokens → reset → typography → layout → components → utilities`。可使用 CSS Cascade Layers 防止全局覆盖失控：

```css
@layer tokens, reset, base, layout, components, utilities;
```

### 12.2 组件边界

```text
src/components/
├── shell/
│   ├── GlobalSidebar
│   ├── KnowledgeSidebar
│   └── ResponsiveDrawer
├── dashboard/
│   ├── QuickActionGrid
│   ├── ActivityTabs

    ├── Button / IconButton

    ├── Tabs / Select / Input
    └── EmptyState / Skeleton / Toast
```

迁移期间允许保留现有组件文件名，但新增样式和子组件必须遵守上述职责；禁止在 `MainArea` 中继续堆叠页面级视觉分支。

### 12.3 兼容与迁移规则

- 先增加新语义令牌，再把旧变量映射到新变量，最后删除旧变量；不能一次性全局替换后再修页面。
- 每迁移一个页面，浅色、深色、1440px 和 390px 四个基线必须同批通过。
- 不修改 R001 的数据 schema；组件拆分只调整展示层与局部 UI 状态。
- 用户已保存的侧栏宽度需做范围归一化：旧值小于 200px 时迁移为 224px。
- UI 改造期间保持键盘快捷键和编辑器命令行为不变。

## 16. 风险与对策

| 风险 | 对策 |
| --- | --- |
| 三栏占用过多导致正文过窄 | 按断点先折叠全局侧栏，再将知识库侧栏改为抽屉 |
| 全局 CSS 拆分造成选择器优先级回归 | 使用 Cascade Layers；按页面迁移并保留短期变量映射 |
| 工具栏改造破坏 Tiptap 命令状态 | 只替换展示层；为每组命令补 active/disabled 组件测试 |
| 深色主题机械反色导致内容块不可读 | 设计独立语义令牌，逐一建立代码、表格、引用、弹层基线 |
| 参考产品过度相似 | 只复用通用布局规律，使用自有主色、名称、图标和文案 |
| 响应式抽屉焦点丢失 | 统一 Drawer 组件，自动锁定和归还焦点，加入 E2E |
| 大目录重绘影响性能 | 保持行样式简单，必要时在后续需求引入虚拟列表 |
